# Contributing to cohive

Thank you for your interest in contributing to **cohive**! We welcome bug reports, feature requests, documentation improvements, and pull requests from the community.

## 🤝 Code of Conduct

Please treat everyone in the community with respect and empathy. We are committed to providing a welcoming, harassment-free environment for everyone.

## 🛠️ Local Development Setup

1. **Fork & Clone** the repository:
   ```bash
   git clone --recursive https://github.com/your-username/cohive-cloudflare.git
   cd cohive-cloudflare
   ```

2. **Install Dependencies**:
   ```bash
   npm install
   ```

3. **Start Development Servers**:
   - Backend Pages Functions API: `npm run pages:dev`
   - Frontend Vite Dev Server: `npm run dev`

## 📝 Submitting Pull Requests

1. Create a descriptive branch name: `git checkout -b feature/awesome-new-feature` or `git checkout -b fix/bug-description`.
2. Keep your changes focused and write clean, readable TypeScript code.
3. Commit your changes with standard commit messages (e.g., `feat: add custom status support`, `fix: resolve D1 query error`).
4. Push to your fork and submit a Pull Request to the `main` branch.

Thank you for helping build an open, lightweight collaboration platform!
